Introduction

JavaScript is a high-level programming language commonly used for creating interactive web pages and web applications. It is a versatile language known for its flexibility and compatibility with various platforms. This documentation provides an overview of JavaScript, including its syntax, data types, control structures, functions, and more..

Syntax

JavaScript syntax is similar to other programming languages such as Java and C. It consists of statements that are executed sequentially or conditionally. Here's a basic syntax example: // This is a single-line comment /* This is a multi-line comment */ // Variables and data types let greeting = 'Hello, world!'; console.log(greeting); // Conditional statement if (greeting === 'Hello, world!') { console.log('The greeting is correct.'); } else { console.log('The greeting is incorrect.'); }

Data types

JavaScript supports various data types, including: Primitive Types: Number, String, Boolean, Undefined, Null, Symbol Composite Types: Object Special Data Types: BigInt let number = 42; let string = 'Hello'; let boolean = true; let undefinedValue = undefined; let nullValue = null; let symbol = Symbol('foo'); let bigInt = 1234567890123456789012345678901234567890n; console.log(typeof number); // Output: number

Variables

In JavaScript, variables are declared using the var, let, or const keyword. Variables declared with var have function-level scope, while those declared with let and const have block-level scope. var x = 10; // Function-level scope let y = 20; // Block-level scope const PI = 3.14; // Constant variable

Opeators

JavaScript supports various operators such as arithmetic, assignment, comparison, logical, bitwise, and more. let a = 10; let b = 5; let result = a + b; // Addition result = a - b; // Subtraction result = a * b; // Multiplication result = a / b; // Division result = a % b; // Modulus result = a ** b; // Exponentiation

Control Structure

JavaScript provides control structures like if-else, switch, for, while, and do-while loops for conditional and iterative execution. let num = 10; if (num > 0) { console.log('Number is positive'); } else if (num < 0) { console.log('Number is negative'); } else { console.log('Number is zero'); }

Functions

Functions in JavaScript are defined using the function keyword. They can be declared as function declarations or function expressions. // Function declaration function greet(name) { return 'Hello, ' + name + '!'; } // Function expression let greet = function(name) { return 'Hello, ' + name + '!'; };

Objects

Objects in JavaScript are collections of key-value pairs. They are used to represent complex data structures and are fundamental to JavaScript programming. let person = { name: 'John', age: 30, isMarried: false, greet: function() { return 'Hello, my name is ' + this.name + '.'; } };

Arrays

Arrays in JavaScript are used to store multiple values in a single variable. They can contain elements of different data types and have various built-in methods for manipulation. let fruits = ['apple', 'banana', 'orange']; fruits.push('mango'); // Add element to the end fruits.pop(); // Remove element from the end

classes

JavaScript introduced class syntax in ECMAScript 6 for object-oriented programming. Classes are blueprints for creating objects with properties and methods. class Rectangle { constructor(width, height) { this.width = width; this.height = height; } area() { return this.width * this.height; } } let rectangle = new Rectangle(5, 10); console.log(rectangle.area()); // Output: 50

Modules

JavaScript modules allow you to organize code into reusable units. They encapsulate related functionality and provide a way to export and import code between files. // Math.js export function add(a, b) { return a + b; } // main.js import { add } from './Math.js'; console.log(add(5, 10)); // Output: 15

Error handling

JavaScript provides try, catch, and finally blocks for error handling. Errors can be thrown using the throw statement. try { // Code that may throw an error throw new Error('An error occurred'); } catch (error) { // Handle the error console.log(error.message); } finally { // Code that runs regardless of whether an error occurred console.log('Cleanup code'); }
